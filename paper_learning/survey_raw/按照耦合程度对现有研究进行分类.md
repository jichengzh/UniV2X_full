# 按耦合程度对现有软硬件协同研究进行分类

> 基于 survey_raw 中 5 个方向共 49 篇论文的分析
> 判断标准：软件决策和硬件决策之间是否存在自动化的联合优化/反馈回路？

---

## 耦合度定义

- **紧耦合**：算法参数和硬件架构/配置在同一优化框架里联合搜索、联合评估，存在自动化反馈
- **松耦合**：先设计好算法（可能人工考虑了硬件特性），再单独做硬件/系统优化，两者之间无自动化闭环
- **中等**：有硬件感知的设计或编译适配，但不是完全的联合搜索

## Agent-1: LLM/Transformer专用加速 — 全部松耦合

| 论文 | 耦合度 | 原因 |
|------|--------|------|
| FlashAttention-2 | 松 | 人工根据GPU SRAM层次设计IO感知算法，无自动化搜索 |
| vLLM | 松 | 借鉴OS分页思想的系统工程，纯软件侧优化 |
| SpecInfer | 松 | 投机解码是算法创新，硬件只是提供批并行能力 |
| SGLang | 松 | 编程框架层面KV cache复用，纯软件侧 |
| Megablocks | 松 | 人工将MoE稀疏转为block-sparse以适配GPU kernel |
| FlashDecoding++ | 松 | 手动针对GPU特性优化解码算法 |
| DistServe | 松 | prefill/decode分到不同GPU实例，人工调度设计 |
| Splitwise | 松 | 异构硬件映射，但映射策略是人工分析得出 |
| Orca | 松 | 连续批处理是调度创新，不涉及硬件参数优化 |
| FlexGen | 松偏中等 | 线性规划自动搜索GPU/CPU/Disk放置策略，有自动化搜索但仅限存储放置一个维度 |

**特点**：这一组全部是"好的系统工程师根据硬件特性设计算法/系统"——人工分析硬件瓶颈，人工设计解决方案，与 EMOS 的模式一致。

| 论文 | 耦合度 | 原因 |
|------|--------|------|
| **MicroScopiQ** | **紧** | 量化+剪枝算法与多精度PE阵列+NoC**联合设计** |
| **Panacea** | **紧** | 非对称量化方案与bit-slice加速器**共同设计** |
| **M-ANT** | **紧** | 自适应数值类型与脉动阵列支持2/4/8-bit**联合设计** |
| **BitMoD** | **紧** | bit-serial混合数据类型与统一加速器**共同设计** |
| AWQ | 松 | 量化算法考虑了硬件友好性，但TinyChat部署是独立的后续步骤 |
| **OliVe** | **紧** | outlier-victim编码格式与硬件解码逻辑**联合设计** |
| **SOFA** | **紧** | 跨阶段tiling调度策略与注意力加速器架构**联合设计** |
| **Phi** | **紧** | 算法端调整稀疏模式 + 硬件端动态生成稀疏模式，**双向适配** |

**特点**：最典型的紧耦合——量化/稀疏方案的设计**直接驱动了专用硬件架构的设计**，反过来硬件的能力也**约束和引导了算法的设计**。两者在同一个设计过程中互相塑造。

| 论文 | 耦合度 | 原因 |
|------|--------|------|
| Souffle | 松 | 编译器自动优化算子融合，但不改变硬件配置 |

| GraphPipe | 松 | 图流水线并行，纯调度优化 |

## Agent-5: 自动驾驶/V2X — 大部分松耦合

| 论文 | 耦合度 | 原因 |
|------|--------|------|
| Q-PETR | 松 | 改造算法使其量化友好，部署是后续独立步骤 |
| QD-BEV | 松 | QAT+蒸馏是纯算法层面优化 |
| **SPADE** | **紧** | **专为点云检测设计ASIC加速器**，算法和硬件联合 |
| PlanKD | 松 | 纯知识蒸馏，无硬件参数介入 |
| QuantV2X | 松 | 全量化+通信压缩，但没有硬件参数联合搜索 |
| PTQAT | 松 | 混合量化策略，纯算法层面 |

## 核心发现

1. **紧耦合集中在"设计新硬件"的工作中**（Agent-2的专用加速器、Agent-4的CIM/Chiplet）——因为设计硬件时天然需要和算法联合考虑

2. **松耦合集中在"用现有硬件"的工作中**（Agent-1的GPU serving、Agent-5的TRT部署）——因为硬件已经固定，优化空间只在软件侧

3. **编译器（Agent-3）处于中间地带**——天然位于软硬件接口处，大多数做"单向感知"（感知硬件特性优化软件），少数做双向适配

4. **我们的方案定位独特**：在**固定硬件**（GPU/Jetson）上做**紧耦合**联合搜索——这正是大部分自动驾驶工作没有做的事情。Agent-5的8篇松耦合论文说明这个gap确实存在

5. **规律总结**：现有紧耦合工作几乎都伴随着**定制硬件设计**（ASIC/FPGA/CIM），在**通用GPU平台**上做紧耦合联合优化的工作非常少，这是一个明确的研究空白
