### 〇.1 全部维度列表

| 维度编号 | 配置维度 | 搜索变量 | 取值范围 | 分类 |
|:--------:|---------|---------|---------|:----:|
| P1 | **FFN 中间维度** | ffn_mid_ratio | {0.4, 0.5, 0.6, 0.7, 0.8, 1.0} | **搜索** |
| P2 | **注意力投影通道** | attn_proj_ratio | {0.0, 0.1, 0.2} | **搜索** |

### 〇.2 维度解耦分析：为什么 P4-P7 不进入搜索空间

> **反思来源**：1.1 可配置量化中，D5 (symmetric)、D6 (scale_method) 被证明与其他维度解耦，
> D5 被 TRT GPU 硬件锁定，D6 在 Phase 0 一次性选定后不再影响搜索。
> 同样的逻辑适用于剪枝的 P4-P7。

#### P4 (importance_criterion) — 与 D6 (calibration method) 完全同构，应锁定

**类比**：D6 控制"用什么方法推导量化 scale"，P4 控制"用什么方法评估通道重要性"。
两者都是**方法选择维度**，其输出（scale / 通道排序）被下游消费，但方法本身不与其他搜索维度产生交互效应。

**量化教训**：D6 实测 minmax vs entropy vs mse 差距仅 0.001-0.002 AMOTA，且可在 Phase 0 一次性选定。
P4 同理：给定相同的剪枝比例 (P1-P3)，不同准则选出的通道集合可能略有不同，但不存在
"Taylor + 30% FFN 在某些层上比 L1 + 30% FFN 有质变"的交互效应。

**硬件视角**：TRT 完全不感知你用 L1 还是 Taylor 选的通道。硬件只看到剪枝后的通道数。

**处理方式**：Phase 0 一次性对比 4 种准则（固定 FFN 30% 剪枝），选最优后锁定。
预计耗时：4 次推理 ≈ 40 分钟。推荐默认值：`taylor`。

#### P5 (pruning_granularity) — 方法策略维度，不影响部署

**类比**：类似量化中"选择 per-tensor 还是 per-channel"的决策。但关键区别在于：
量化的 D3/D4 粒度直接影响 TRT 的 Q/DQ scale tensor 形状和 kernel 选择（**紧耦合**）；
而剪枝的 global/local/isomorphic 只影响"跨层如何排序通道重要性"，
最终模型结构一旦确定，硬件完全不感知你用的是 global 还是 local 排序。

**处理方式**：Phase 0 一次性对比（固定 FFN 30% 剪枝），选最优后锁定。
预计耗时：3 次推理 ≈ 30 分钟。推荐默认值：`local`。

#### P6 (iterative_steps) — 算法超参数，非结构变量

**类比**：相当于量化中"entropy 校准用 2048 个 bin 还是 4096 个 bin"——没有人把这种
算法内部超参数放进搜索空间。更多步骤通常更稳定，但不改变最终模型结构的搜索语义。

**处理方式**：锁定为经验值 5。若 Phase 0 发现 iterative_steps=1 与 =5 精度差 < 0.002，
可降为 1 以加速后续搜索。

#### P7 (round_to) — 硬约束，不是搜索变量

**量化教训**：`ch % 8 != 0` 时 TRT 自动 fallback 到 FP32，导致"剪了反而更慢"。
这与 D5 (symmetric) 完全类比——TRT GPU 硬件只支持对称量化，没得选；
同理，INT8 部署下 `round_to=8` 是唯一正确选择，不是可以权衡的维度。

**处理方式**：锁定为 8（INT8 部署硬约束）。作为 `constraints.channel_alignment` 写入配置。

### 〇.3 修正后的搜索空间

```
锁定维度 (Phase 0 一次性确定，不进入联合搜索):
  P4 importance_criterion  →  Phase 0 对比后选最优（推荐 taylor）
  P5 pruning_granularity   →  Phase 0 对比后选最优（推荐 local）
  P6 iterative_steps       →  锁定 5（经验值）
  P7 round_to              →  锁定 8（INT8 硬约束）

搜索维度 (进入 B1×B2×D 联合搜索空间):
  P1 ffn_mid_ratio         ∈ {0.4, 0.5, 0.6, 0.7, 0.8, 1.0}  (6 值)
  P2 attn_proj_ratio       ∈ {0.0, 0.1, 0.2}                   (3 值)
  P3 head_mid_ratio        ∈ {0.5, 0.7, 1.0}                   (3 值)
  P8 head_pruning_ratio    ∈ {0.0, 0.125}                       (2 值)
  P9 decoder_num_layers    ∈ {5, 6}                              (2 值)

搜索空间大小: 6 × 3 × 3 × 2 × 2 = 216 个配置
(vs 原始 P1-P9 全搜索: 8×5×4×4×3×4×3×3×3 = 155,520 个 → 缩减 720x)
```

> 注：P1-P3 直接改变计算量，与硬件 kernel 选择紧耦合；P8-P9 改变模型拓扑，影响 TRT graph 结构。
> 这 5 个维度是剪枝搜索空间中真正与量化(B2)和部署(D)产生交互效应的维度。
> P1 优先级最高（FFN 占 Transformer 参数 ~40%），P9 优先级最低（层数剪枝需要蒸馏恢复）。
> sampling_offsets 和 attention_weights 层为坐标敏感层，**硬约束：不可剪枝**。

```
  │   ├─ attention_weights  Linear(256→...)     ← 硬约束: 不剪枝

检测头 (cls_branches × 7 + reg_branches × 7):
  ├─ Linear(256→256) × N                        ← P3 控制

```

> 目标：基于 Torch-Pruning (DepGraph) 框架，使搜索维度 (P1-P3, P8-P9) 可配置并可快速评估精度；

### A.1 当前 PyTorch 侧代码能力审计

| 维度 | 分类 | 代码位置 | 当前状态 | 缺口 |
|:----:|:----:|---------|:-------:|------|

锁定维度 (P4-P7) 仍需实现以支持 Phase 0 对比，但不进入后续联合搜索。

### A.2 需要实现的内容

#### A.2.1 DepGraph 适配层：自定义剪枝器

**新建文件**: `projects/mmdet3d_plugin/univ2x/pruning/custom_pruners.py`

**问题**: UniV2X 使用自定义 CUDA 算子 (`MultiScaleDeformableAttnFunction`)，DepGraph 的 `torch.fx` 追踪无法穿透这些算子。此外，SCA 中的 `nonzero()` 动态索引也会阻断追踪。

**解决方案**: 为不可追踪的模块注册自定义剪枝器，告诉 DepGraph 这些模块的输入/输出通道关系。

```python
import torch_pruning as tp

class MSDeformableAttention3DPruner(tp.BasePruningFunc):
    """自定义剪枝器：处理 MSDeformableAttention3D 模块"""
    
    def prune_out_channels(self, layer, idxs):
        # MSDeformableAttention3D 的输出通道由 output_proj 控制
        # value_proj 的输出通道与 embed_dims 绑定
        # sampling_offsets 和 attention_weights 不参与通道剪枝
        
        # 剪枝 value_proj 的输出通道
        tp.prune_linear_out_channels(layer.value_proj, idxs)
        
        # 不动 sampling_offsets 和 attention_weights
        # 它们的维度由 num_heads * num_levels * num_points 决定，与 embed_dims 无关
        
    def prune_in_channels(self, layer, idxs):
        # 输入通道来自上游特征，影响 value_proj 的输入
        tp.prune_linear_in_channels(layer.value_proj, idxs)
        # sampling_offsets 和 attention_weights 的输入也来自 query
        # 但因为它们是坐标敏感层，不剪枝
        # 注意：这里需要特殊处理 —— 如果上游 embed_dims 被剪了，
        # sampling_offsets 的输入也需要对应调整，但输出维度不变
        tp.prune_linear_in_channels(layer.sampling_offsets, idxs)
        tp.prune_linear_in_channels(layer.attention_weights, idxs)
    
    def get_out_channels(self, layer):
        return layer.value_proj.out_features
    
    def get_in_channels(self, layer):
        return layer.value_proj.in_features


class SpatialCrossAttentionPruner(tp.BasePruningFunc):
    """自定义剪枝器：处理 SpatialCrossAttention 模块"""
    
    def prune_out_channels(self, layer, idxs):
        tp.prune_linear_out_channels(layer.output_proj, idxs)
        # 递归处理内部 deformable_attention
    
    def prune_in_channels(self, layer, idxs):
        tp.prune_linear_in_channels(layer.output_proj, idxs)
    
    def get_out_channels(self, layer):
        return layer.output_proj.out_features
    
    def get_in_channels(self, layer):
        return layer.output_proj.in_features


class TemporalSelfAttentionPruner(tp.BasePruningFunc):
    """自定义剪枝器：处理 TemporalSelfAttention 模块"""
    
    def prune_out_channels(self, layer, idxs):
        tp.prune_linear_out_channels(layer.output_proj, idxs)
        tp.prune_linear_out_channels(layer.value_proj, idxs)
    
    def prune_in_channels(self, layer, idxs):
        # TSA 输入是 concat(prev_bev, current_bev)，维度是 embed_dims*2
        # 需要特殊处理：将 idxs 映射到双倍维度
        double_idxs = []
        for i in idxs:
            double_idxs.extend([i, i + layer.embed_dims])
        tp.prune_linear_in_channels(layer.sampling_offsets, double_idxs)
        tp.prune_linear_in_channels(layer.attention_weights, double_idxs)
        tp.prune_linear_in_channels(layer.value_proj, idxs)
    
    def get_out_channels(self, layer):
        return layer.output_proj.out_features
    
    def get_in_channels(self, layer):
        return layer.value_proj.in_features
```

**注册自定义剪枝器**:

```python
def register_univ2x_pruners():
    """注册 UniV2X 特有模块的自定义剪枝器"""
    from ..modules.spatial_cross_attention import (
        SpatialCrossAttention, MSDeformableAttention3D)
    from ..modules.temporal_self_attention import TemporalSelfAttention
    from ..modules.decoder import CustomMSDeformableAttention
    
    return {
        MSDeformableAttention3D: MSDeformableAttention3DPruner(),
        SpatialCrossAttention: SpatialCrossAttentionPruner(),
        TemporalSelfAttention: TemporalSelfAttentionPruner(),
        CustomMSDeformableAttention: MSDeformableAttention3DPruner(),
    }
```

**重要性**: 这是整个剪枝方案最核心也最有风险的部分。自定义剪枝器必须精确描述每个模块的通道依赖关系，否则剪枝后模型维度不匹配会直接崩溃。

#### A.2.2 剪枝主入口

**新建文件**: `projects/mmdet3d_plugin/univ2x/pruning/prune_univ2x.py`

**核心逻辑**:

```python
import torch
import torch_pruning as tp

def build_dependency_graph(model, example_inputs, prune_cfg):
    """构建 DepGraph 依赖图"""
    
    customized_pruners = register_univ2x_pruners()
    
    # 收集不可剪枝的层
    ignored_layers = _collect_ignored_layers(model, prune_cfg)
    
    # 收集 unwrapped parameters（位置编码等非 Module 参数）
    unwrapped_parameters = _collect_unwrapped_params(model)
    
    DG = tp.DependencyGraph()
    DG.build_dependency(
        model,
        example_inputs=example_inputs,
        customized_pruners=customized_pruners,
        ignored_layers=ignored_layers,
        unwrapped_parameters=unwrapped_parameters,
    )
    return DG


def build_pruner(model, example_inputs, prune_cfg):
    """构建剪枝器
    
    锁定维度 (Phase 0 确定后固定):
      P4 importance_criterion → locked_config.importance (默认 taylor)
      P5 pruning_granularity  → locked_config.granularity (默认 local)
      P6 iterative_steps      → 固定 5
      P7 round_to             → 固定 8 (INT8 硬约束)
    
    搜索维度 (从 prune_cfg 读取):
      P1 ffn_mid_ratio, P2 attn_proj_ratio, P3 head_mid_ratio
      P8 head_pruning_ratio, P9 decoder_num_layers
    """
    locked = prune_cfg.get('locked', {})
    
    # 锁定维度：Phase 0 确定后不再变动
    importance = _select_importance(
        locked.get('importance_criterion', 'taylor'))            # P4 锁定
    
    # 搜索维度：P1-P3 构建逐层剪枝比例
    pruning_ratio_dict = _build_ratio_dict(model, prune_cfg)
    
    # 搜索维度：P8 收集注意力头信息
    num_heads = _collect_num_heads(model)
    
    pruner = tp.pruner.BasePruner(
        model,
        example_inputs,
        importance=importance,
        pruning_ratio_dict=pruning_ratio_dict,
        global_pruning=locked.get('granularity', 'local') == 'global',  # P5 锁定
        isomorphic=locked.get('granularity', 'local') == 'isomorphic',  # P5 锁定
        iterative_steps=locked.get('iterative_steps', 5),               # P6 锁定
        round_to=locked.get('round_to', 8),                             # P7 锁定
        ignored_layers=_collect_ignored_layers(model, prune_cfg),
        customized_pruners=register_univ2x_pruners(),
        num_heads=num_heads,
        head_pruning_ratio=prune_cfg.get('head_pruning_ratio', 0.0),    # P8 搜索
    )
    return pruner


def _select_importance(criterion):
    """P4: 选择重要性评估方法"""
    mapping = {
        'l1_norm': tp.importance.GroupMagnitudeImportance(p=1),
        'l2_norm': tp.importance.GroupMagnitudeImportance(p=2),
        'taylor': tp.importance.GroupTaylorImportance(),
        'bn_scale': tp.importance.BNScaleImportance(),
        'fpgm': tp.importance.FPGMImportance(),
        'hessian': tp.importance.GroupHessianImportance(),
        'random': tp.importance.RandomImportance(),
    }
    if criterion not in mapping:
        raise ValueError(f"未知的重要性准则: {criterion}, 可选: {list(mapping.keys())}")
    return mapping[criterion]


def _build_ratio_dict(model, prune_cfg):
    """P1-P3: 为不同模块类型设置剪枝比例"""
    ratio_dict = {}
    
    for name, module in model.named_modules():
        if not isinstance(module, torch.nn.Linear):
            continue
        
        # P1: FFN 中间层
        if _is_ffn_layer(name):
            ratio_dict[module] = 1.0 - prune_cfg.get('ffn_mid_ratio', 1.0)
        
        # P2: 注意力投影层
        elif _is_attn_proj(name):
            ratio_dict[module] = prune_cfg.get('attn_proj_ratio', 0.0)
        
        # P3: 检测头中间层
        elif _is_head_layer(name):
            ratio_dict[module] = 1.0 - prune_cfg.get('head_mid_ratio', 1.0)
    
    return ratio_dict


def _is_ffn_layer(name):
    """判断是否为 FFN 中间层"""
    return any(k in name for k in ['ffns.', 'feedforward.'])


def _is_attn_proj(name):
    """判断是否为注意力投影层（非坐标敏感）"""
    return any(k in name for k in ['value_proj', 'output_proj'])


def _is_head_layer(name):
    """判断是否为检测头中间层（非最终输出层）"""
    is_head = any(k in name for k in ['cls_branches', 'reg_branches'])
    is_final = name.endswith('.weight') or 'final' in name
    return is_head and not is_final


def _collect_ignored_layers(model, prune_cfg):
    """收集不可剪枝的层"""
    ignored = []
    for name, module in model.named_modules():
        # 硬约束：坐标敏感层
        if any(k in name for k in ['sampling_offsets', 'attention_weights']):
            ignored.append(module)
        # 硬约束：最终输出层
        if _is_output_layer(name, module):
            ignored.append(module)
    return ignored


def _is_output_layer(name, module):
    """判断是否为最终输出层（不可剪枝）"""
    # 分类头最后一层、回归头最后一层
    if isinstance(module, torch.nn.Linear):
        if any(k in name for k in ['cls_branches.6', 'reg_branches.6']):
            return True
    return False


def _collect_num_heads(model):
    """收集注意力模块的头数信息"""
    num_heads = {}
    for name, module in model.named_modules():
        if hasattr(module, 'num_heads') and hasattr(module, 'embed_dims'):
            num_heads[module] = module.num_heads
    return num_heads


def _collect_unwrapped_params(model):
    """收集位置编码等非 Module 的可学习参数"""
    unwrapped = []
    for name, param in model.named_parameters():
        if 'bev_embedding' in name or 'query_embedding' in name:
            unwrapped.append((param, 1))  # 在 dim=1 (embed_dim) 上剪枝
    return unwrapped
```

#### A.2.3 P9：解码器层数剪枝

**修改文件**: `projects/mmdet3d_plugin/univ2x/pruning/prune_univ2x.py`

**当前问题**: 层数剪枝不属于通道剪枝，DepGraph 不直接支持，需要手动实现。

**实现要点**:
- 直接从 `decoder.layers` 中删除指定层
- 同步删除对应的迭代预测分支 (`cls_branches[i]`, `reg_branches[i]`)
- 更新 `decoder.num_layers` 属性
- 需要重新对齐剩余层的 reference_points 预测

```python
def prune_decoder_layers(model, target_num_layers):
    """P9: 删除解码器层"""
    decoder = model.pts_bbox_head.transformer.decoder
    current_layers = len(decoder.layers)
    
    if target_num_layers >= current_layers:
        return model  # 无需剪枝
    
    # 策略：保留最后 N 层（最后几层通常对精度贡献最大）
    keep_indices = list(range(current_layers - target_num_layers, current_layers))
    
    # 删除层
    new_layers = torch.nn.ModuleList([decoder.layers[i] for i in keep_indices])
    decoder.layers = new_layers
    decoder.num_layers = target_num_layers
    
    # 同步删除迭代预测分支
    head = model.pts_bbox_head
    if hasattr(head, 'cls_branches'):
        head.cls_branches = torch.nn.ModuleList(
            [head.cls_branches[i] for i in keep_indices] + [head.cls_branches[-1]])
    if hasattr(head, 'reg_branches'):
        head.reg_branches = torch.nn.ModuleList(
            [head.reg_branches[i] for i in keep_indices] + [head.reg_branches[-1]])
    
    return model
```

#### A.2.4 梯度计算（供 Taylor/Hessian 重要性使用）

**新建文件**: `projects/mmdet3d_plugin/univ2x/pruning/grad_collector.py`

**功能**: 用少量校准数据计算梯度，供 `GroupTaylorImportance` 使用。

```python
def collect_gradients(model, dataloader, num_batches=32):
    """
    收集梯度信息，供 Taylor 重要性评估使用。
    
    复用现有校准数据管线 (calibration/*.pkl) 或直接使用 dataloader。
    """
    model.train()  # 需要启用梯度
    model.requires_grad_(True)
    
    for batch_idx, data in enumerate(dataloader):
        if batch_idx >= num_batches:
            break
        
        # UniV2X 的 forward 返回 loss dict
        losses = model.train_step(data, optimizer=None)
        loss = sum(v for k, v in losses.items() 
                   if 'loss' in k and isinstance(v, torch.Tensor))
        loss.backward()
    
    # 注意：不清零梯度，让 DepGraph 在后续步骤中读取累积梯度
```

#### A.2.5 剪枝后模块状态更新

**新建文件**: `projects/mmdet3d_plugin/univ2x/pruning/post_prune.py`

**功能**: DepGraph 剪枝后，部分模块的内部属性（如 `num_heads`, `head_dim`, `embed_dims`）不会自动更新，需要手动修正。

```python
def update_model_after_pruning(model):
    """剪枝后更新模块内部状态"""
    
    for name, module in model.named_modules():
        # 更新注意力模块
        if hasattr(module, 'embed_dims') and hasattr(module, 'value_proj'):
            new_embed_dims = module.value_proj.out_features
            module.embed_dims = new_embed_dims
            if hasattr(module, 'head_dim') and hasattr(module, 'num_heads'):
                module.head_dim = new_embed_dims // module.num_heads
        
        # 更新 FFN 模块
        if hasattr(module, 'feedforward_channels'):
            for sub_name, sub_module in module.named_modules():
                if isinstance(sub_module, torch.nn.Linear) and 'layers.0' in sub_name:
                    module.feedforward_channels = sub_module.out_features
                    break
        
        # 更新 BEVFormerEncoder 的 embed_dims
        if hasattr(module, 'embed_dims') and hasattr(module, 'layers'):
            # 从第一个 FFN 的输出维度推断
            for layer in module.layers:
                if hasattr(layer, 'ffns'):
                    last_linear = list(layer.ffns[0].modules())[-2]  # 最后一个 Linear
                    if isinstance(last_linear, torch.nn.Linear):
                        module.embed_dims = last_linear.out_features
                    break
    
    # 更新 LayerNorm 维度（DepGraph 通常自动处理，但验证一下）
    for name, module in model.named_modules():
        if isinstance(module, torch.nn.LayerNorm):
            # LayerNorm 的 normalized_shape 应与上游输出匹配
            actual_dim = module.weight.shape[0]
            module.normalized_shape = (actual_dim,)
```

```python
def apply_prune_config(model, prune_config, dataloader=None):
    """将 prune_config 应用到模型
    
    搜索维度 (P1-P3, P8-P9): 从配置中读取，每次搜索可变
    锁定维度 (P4-P7): 从 locked 段读取，Phase 0 确定后固定
    """
    locked = prune_config.get('locked', {})
    enc = prune_config.get('encoder', {})
    dec = prune_config.get('decoder', {})
    heads = prune_config.get('heads', {})
    
    # 0. 如果使用 Taylor/Hessian 准则（锁定维度 P4），先收集梯度
    criterion = locked.get('importance_criterion', 'taylor')
    if criterion in ('taylor', 'hessian') and dataloader:
        collect_gradients(model, dataloader, num_batches=32)
    
    # 1. 构建示例输入（用于 DepGraph 追踪）
    example_inputs = _build_example_inputs(model)
    
    # 2. 构建 prune_cfg（搜索维度 + 锁定维度）
    prune_cfg = {
        # 锁定维度 (Phase 0 确定)
        'locked': locked,
        # 搜索维度
        'ffn_mid_ratio': enc.get('ffn_mid_ratio', 1.0),                # P1
        'attn_proj_ratio': enc.get('attn_proj_ratio', 0.0),            # P2
        'head_mid_ratio': heads.get('head_mid_ratio', 1.0),            # P3
        'head_pruning_ratio': enc.get('head_pruning_ratio', 0.0),      # P8
    }
    
    # 3. 执行通道剪枝 (P1-P3, P8)
    pruner = build_pruner(model, example_inputs, prune_cfg)
    for step in range(locked.get('iterative_steps', 5)):
        pruner.step()
    
    # 4. 更新模块内部状态
    update_model_after_pruning(model)
    
    # 5. 执行层数剪枝 (P9 搜索维度)
    target_layers = dec.get('num_layers', 6)
    model = prune_decoder_layers(model, target_layers)
    
    return model
```

**新建文件**: `tools/verify_trt_constraints.py`

**功能**: 在 TRT 构建前，验证剪枝后的模型是否满足所有硬件约束。

```python
def verify_hw_constraints(pruned_model, prune_config):
    """验证剪枝后模型的硬件部署约束"""
    
    violations = []
    round_to = prune_config['constraints']['channel_alignment']
    
    for name, module in pruned_model.named_modules():
        if isinstance(module, (torch.nn.Linear, torch.nn.Conv2d)):
            out_ch = module.out_features if isinstance(module, torch.nn.Linear) \
                     else module.out_channels
            in_ch = module.in_features if isinstance(module, torch.nn.Linear) \
                    else module.in_channels
            
            # 约束 1: 通道对齐
            if out_ch % round_to != 0:
                violations.append(
                    f"[ALIGN] {name}: out_channels={out_ch} 不是 {round_to} 的倍数")
            
            # 约束 2: 最小通道数
            min_ch = prune_config['constraints']['min_channels']
            if out_ch < min_ch:
                violations.append(
                    f"[MIN_CH] {name}: out_channels={out_ch} < 最小值 {min_ch}")
        
        # 约束 3: 注意力头对齐
        if hasattr(module, 'num_heads') and hasattr(module, 'embed_dims'):
            if module.embed_dims % module.num_heads != 0:
                violations.append(
                    f"[HEAD] {name}: embed_dims={module.embed_dims} "
                    f"不能被 num_heads={module.num_heads} 整除")
        
        # 约束 4: 残差连接维度匹配
        # (DepGraph 自动保证，但做 double check)
    
    if violations:
        print("⚠️ 硬件约束违反:")
        for v in violations:
            print(f"  {v}")
        return False
    
    print("✅ 所有硬件约束满足")
    return True
```

## 附录：与 1.1 可配置量化的对比

| 对比项 | 1.1 可配置量化 (B2) | 1.2 可配置剪枝 (B1) |
|--------|:------------------:|:------------------:|
| 全部维度 | D1-D9 (9 维) | P1-P9 (9 维) |
| **搜索维度** | **D1-D4, D7-D8 (6 维)** | **P1-P3, P8-P9 (5 维)** |
| **锁定维度** | **D5, D6, D9 (3 维)** | **P4, P5, P6, P7 (4 维)** |
| 锁定原因 | D5=硬件约束, D6=解耦, D9=解耦 | P4/P5=解耦, P6=超参数, P7=硬件约束 |
| 核心框架 | 自研 QuantModel + UniformAffineQuantizer | Torch-Pruning (DepGraph) |
| 修改模型权重 | 否 (fake-quant, 只加 scale/zp) | 是 (物理删除通道/层) |
| 需要微调 | 否 (PTQ) | 是 (10 epoch, 必需) |
| TRT 落地 | Q/DQ 显式模式 或 隐式 Calibrator | 重新导出 ONNX → 隐式/显式均可 |
| 校准数据 | 可复用 | 必须重新收集 (shape 变了) |
| 精度恢复 | 自动 (校准即恢复) | 需要微调 (剪枝后精度下降更大) |
| 加速机制 | 低精度计算 (INT8 kernel) | 减少计算量 (更少通道/层) |
| 4090 加速上限 | ~2x (INT8) | ~1.5x (30% 剪枝) |
| 联合加速 | — | ~3x (剪枝 + INT8) |
| 最大风险 | Q/DQ 注入失败 (已有经验) | CUDA 自定义算子追踪失败 |

## 附录：风险清单与缓解措施

| 风险 | 等级 | 缓解措施 |
|------|:----:|---------|
| CUDA 自定义算子 (MSDA) 阻断 DepGraph 追踪 | **高** | 自定义剪枝器 (A.2.1) 绕过；先用 pure-PyTorch 实现验证 |
| TSA 时序状态 `prev_bev` 维度不匹配 | **高** | TSA 的 embed_dims 被剪后，prev_bev cache 也需同步变化；在推理 hook 中处理 |
| 残差连接两侧维度不匹配 | **中** | DepGraph 自动保证；但自定义剪枝器可能遗漏，需 A.2.5 状态更新兜底 |
| V2X 融合模块 ego/infra 维度不一致 | **中** | ego 和 infra 使用相同 prune_config；在 B.5 硬件约束验证中检查 |
| 剪枝 + 量化叠加精度损失超预期 | **中** | 先单独验证各自损失，再联合；目标: 联合 < 单独之和 |
| 检测头分支间权重共享被破坏 | **低** | cls_branches/reg_branches 7 层迭代预测共享结构，剪枝时保持一致 |
| 微调数据不足 | **低** | 复用 NuScenes 训练集，10 epoch 足够恢复 |
| DepGraph 版本兼容性 | **低** | 锁定 torch-pruning==1.4.x，在 requirements.txt 中固定 |
