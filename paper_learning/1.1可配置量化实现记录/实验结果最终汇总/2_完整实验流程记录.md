## 一、Phase A1 — 量化器核心增强

### 1.1 目标
为 `UniformAffineQuantizer` 补齐三项关键缺口：对称量化 (D5)、entropy+channel-wise (D4)、percentile 校准 (D6)。

### 1.2 实现内容

**对称量化 (D5)**：
- 修改 `quant_layer.py` 的 `__init__`、`forward`、`calculate_qparams`、`quantize`、`perform_2D_search`、`bitwidth_refactor` 共 6 处
- 对称量化范围：[-127, 127] for INT8，zero_point 始终为 0
- 此前第 61 行 `raise NotImplementedError` 阻断了对称量化

**entropy + channel-wise (D4)**：
- 提取 `_entropy_search_1d` 辅助方法，逐通道循环调用
- 此前第 226 行 `raise NotImplementedError` 阻断了 channel-wise entropy

**percentile 校准 (D6)**：
- 新增 `perform_percentile_search`，使用 `torch.quantile` 裁剪离群值
- `get_x_min_x_max` 增加 percentile 分支

### 1.3 测试结果

8 个单元测试全部通过：
- 对称量化基本正确性 (minmax)
- 对称 + channel-wise
- 对称 vs 非对称误差对比：对称 0.007356 vs 非对称 0.005885（对称大约 25%）
- entropy + channel-wise
- percentile (含离群值)
- bitwidth_refactor (4bit: -7/7, 6bit: -31/31)
- percentile + channel-wise
- MSE + 对称 (err=0.005384，优于 minmax)

### 1.4 发现

- entropy + channel-wise 极慢：32 通道 x 2048 bins = 6+ 分钟，不适合搜索循环
- 校准方法排序：mse(0.005384) < entropy(0.005790) < minmax(0.007356)
- 建议：搜索阶段用 minmax（最快），Top-K 用 mse（最精确）

## 二、Phase A2 — Per-group 量化

### 2.1 目标
在 `UniformAffineQuantizer` 中新增 per-group 量化粒度 (D3)。

### 2.2 实现

- 新增 `group_size` 和 `use_group_quant` 参数
- forward 中新增 `_quantize_per_group` 方法：reshape → 按组量化 → reshape 回
- 例：Linear(256, 1024), group_size=64 → 4096 个 scale 值

### 2.3 测试结果

5 个测试全部通过。精度排序验证正确：
```
per-group(32): err=0.004472
per-group(64): err=0.005024
per-channel:   err=0.005956
per-tensor:    err=0.008172
```

INT4+per-group(64): err=0.091371（有意义的组合 — INT4 位宽损失被 per-group 补偿）

## 四、Phase A4 — 敏感度分析

### 4.1 目标
对 D1-D9 全部维度采集基线数据，缩减搜索空间。

### 4.2 工具实现

新建 `tools/sensitivity_analysis.py`（1040 行），6 个分析模式：
- layer_sensitivity：逐层 INT8 开关 → BEV cosine similarity
- calibration：mse / minmax / entropy / percentile 对比
- granularity：per-tensor / per-channel 对比
- symmetry：对称 vs 非对称
- bitwidth：INT4 / INT6 / INT8 对比
- interaction：层对联合量化的交互项

**设计决策**：使用 BEV cosine similarity 代替 AMOTA（3 秒 vs 3 分钟，快 60 倍）。

### 4.3 GPU 执行结果

**逐层敏感度（54 层，~90s）**：
- 45 safe_int8 (83%), 9 search (17%), **0 skip_fp16**
- 结论：BEV encoder 对逐层量化容忍度极高

**search 层规律**：
- 后层 (3/4/5) 比前层 (0/1/2) 敏感
- SCA 比 TSA 敏感
- FFN output 比 hidden 敏感

**交互项分析**：
- 3 对关键层对的交互项全部为负 → 层间误差相互抵消
- 线性加性假设安全

**全模型对比分析（calibration/granularity/symmetry/bitwidth）**：
- 全部 cos=1.000000 → 被 45 个不敏感层稀释，分辨力不足
- 改进方向：应只测量 search 层的输出 cosine

## 五、Phase B1 — Q/DQ 注入工具

### 5.1 目标
将 PyTorch quant_config 中的 D1-D8 映射到 ONNX Q/DQ 节点。

### 5.2 实现

新建 `tools/inject_qdq_from_config.py`（767 行）：
- 支持 3 种 pattern matching（Constant+Transpose+MatMul, Initializer+Transpose+MatMul, Initializer+MatMul）
- 权重侧 Q/DQ：per-tensor / per-channel scale
- 激活侧 Q/DQ：per-tensor scale，含共享激活输入的去重处理
- 位宽模拟：INT4(max/7), INT6(max/31), INT8(max/127)
- W-only 特殊处理：透明激活 Q/DQ (large scale passthrough)
- 拓扑排序维护：权重 Q/DQ 紧随源节点，激活 Q/DQ 在消费 MatMul 之前

### 5.3 GPU 验证结果

真实 ONNX 上验证通过：**60 weight Q/DQ + 43 activation Q/DQ = 206 节点**。

### 5.4 Bug 修复

1. `a_scale` list 类型 → isinstance 检查后转 np.array
2. weight scale 维度不匹配 → 从 ONNX 权重实际形状重新计算 per-channel scale

### 5.5 与之前失败方案的关键区别

| 维度 | 失败方案 (build_qdq_onnx_adaround.py) | 本次方案 |
|---|---|---|
| 激活 Q/DQ | **无（根因）** | **有，含去重** |
| 对称性 | 非对称 (TRT 不支持) | 对称 (zp=0) |
| Scale 来源 | AdaRound (非对称) | PyTorch 校准 (对称) |
| 粒度 | 固定 per-channel | per-tensor / per-channel 可配 |
| 位宽 | 固定 INT8 | INT4/6/8 通过 scale 模拟 |
| 层级控制 | 全部统一 | 逐层差异化 |
| 依赖 | QuantModel + AdaRound | **纯 ONNX + numpy** |
