### A.1 当前 PyTorch 侧代码能力审计

| 维度 | 代码位置 | 当前状态 | 缺口 |

### A.2 需要补实现的内容

#### A.2.1 对称量化（D5）

**修改文件**: `quant_layer.py` 的 `UniformAffineQuantizer`

**当前问题**: 第 61 行 `if self.sym: raise NotImplementedError`

**修改要点**:
- 对称量化范围: [-2^(n-1)+1, 2^(n-1)-1]，即 INT8 时 [-127, 127]
- scale = max(|x_min|, |x_max|) / qmax
- zero_point 始终为 0
- forward: x_q = clamp(round(x / scale), -qmax, qmax); x_dq = x_q * scale
- calculate_qparams / perform_1D_search / perform_2D_search 均需适配

**重要性**: 这是最关键的缺口。TRT GPU 要求对称 INT8，PyTorch 侧不支持对称就无法与 TRT 行为对齐，敏感度分析结果可能有系统性偏差。

---

#### A.2.2 entropy + channel-wise 支持（D4）

**修改文件**: `quant_layer.py` 的 `perform_entropy_search`

**当前问题**: 第 226 行 `if self.channel_wise: raise NotImplementedError`

**修改要点**:
- 对每个输出通道独立做 histogram + KL divergence 搜索
- 外层遍历通道维度，内层复用现有单通道 entropy 逻辑

#### A.2.3 per-group 量化粒度（D3）

**修改文件**: `quant_layer.py` 的 `UniformAffineQuantizer`

**当前状态**: 只有 per-tensor（channel_wise=False）和 per-channel（channel_wise=True）

**新增 per-group**:
- 新增参数 `group_size: int`（如 64、128）
- 将权重按 group_size 分组，每组独立计算 scale
- 例如 Linear(256, 1024)，group_size=64 时产生 256x(1024/64)=4096 个 scale 值
- forward 中 reshape -> 按组量化 -> reshape 回原始形状

**参考**: FGMP (NVIDIA) 的 block-level mixed-precision 实现

---

#### A.2.4 percentile 校准方法（D6）

**修改文件**: `quant_layer.py` 的 `UniformAffineQuantizer`

**当前状态**: `get_x_min_x_max` 支持 mse/minmax/entropy

**新增 percentile**:
```python
def perform_percentile_search(self, x, percentile=99.99):
    if self.channel_wise:
        y = torch.flatten(x, 1)
        x_min = torch.quantile(y, (100 - percentile) / 100, dim=1)
        x_max = torch.quantile(y, percentile / 100, dim=1)
    else:
        x_min = torch.quantile(x, (100 - percentile) / 100)
        x_max = torch.quantile(x, percentile / 100)
    return x_min, x_max
```

#### A.2.5 V2X 通信特征量化（D9）

**当前状态**: V2X 通信张量（AgentQueryFusion、LaneQueryFusion、BEV scatter）在 PyTorch 中以 FP32/FP16 传输，不在 QuantModel 的量化范围内。

**实现方案**: 不修改 QuantModel 框架，而是在通信点处独立插入量化/反量化：

```python
class CommQuantizer(nn.Module):
    """V2X 通信特征量化器，模拟通信带宽压缩"""
    def __init__(self, n_bits=8, symmetric=True):
        super().__init__()
        self.quantizer = UniformAffineQuantizer(
            n_bits=n_bits, symmetric=symmetric, 
            channel_wise=False, scale_method='minmax')
    
    def forward(self, x):
        # 发送端量化（模拟带宽压缩）
        x_q = self.quantizer(x)
        # 接收端反量化（自动由 fake-quant 完成）
        return x_q
```

插入位置：
- `AgentQueryFusionTRT.forward_trt()` 中 infra query 传入 ego 之前
- `LaneQueryFusionTRT.forward_trt()` 中 infra lane_query 传入 ego 之前
- `_get_coop_bev_embed` 中 infra BEV 特征传入 ego 之前

**apply_quant_config 逻辑**:

```python
def apply_quant_config(model, quant_config):
    """将 quant_config 中的全部维度 (D1-D9) 应用到模型"""
    
    g = quant_config['global']
    
    # 1. QuantModel 包裹（使用 global 默认参���）
    weight_params = {
        'n_bits': g['default_w_bits'],
        'symmetric': g['symmetric'],
        'channel_wise': g['default_w_granularity'] == 'per_channel',
        'scale_method': g['scale_method'],
    }
    act_params = {
        'n_bits': g['default_a_bits'],
        'symmetric': g['symmetric'],
        'channel_wise': g['default_a_granularity'] == 'per_channel',
        'scale_method': g['scale_method'],
    }
    qmodel = QuantModel(model, weight_params, act_params)
    
    # 2. 逐层覆盖 (D1-D8)
    for name, module in qmodel.model.named_modules():
        if not isinstance(module, QuantModule):
            continue
        
        cfg = quant_config['layers'].get(name, {})
        
        # D8: quant_target
        target = cfg.get('quant_target', g['default_quant_target'])
        if target == 'none':
            module.set_quant_state(False, False)
            continue
        module.set_quant_state(
            weight_quant='W' in target,
            act_quant='A' in target
        )
        
        # D1: w_bits
        if 'w_bits' in cfg:
            module.weight_quantizer.bitwidth_refactor(cfg['w_bits'])
        
        # D2: a_bits
        if 'a_bits' in cfg:
            module.act_quantizer.bitwidth_refactor(cfg['a_bits'])
        
        # D3: w_granularity
        w_gran = cfg.get('w_granularity', g['default_w_granularity'])
        if w_gran == 'per_channel':
            module.weight_quantizer.channel_wise = True
        elif w_gran == 'per_group':
            module.weight_quantizer.channel_wise = False
            module.weight_quantizer.group_size = cfg.get('w_group_size', 128)
            module.weight_quantizer.use_group_quant = True
        else:
            module.weight_quantizer.channel_wise = False
        
        # D4: a_granularity
        a_gran = cfg.get('a_granularity', g['default_a_granularity'])
        module.act_quantizer.channel_wise = (a_gran == 'per_channel')
        
        # D5: symmetric (已在 global 设置，逐层如需覆盖可扩展)
        
        # D6: scale_method (逐层覆盖)
        if 'scale_method' in cfg:
            module.weight_quantizer.scale_method = cfg['scale_method']
            module.act_quantizer.scale_method = cfg['scale_method']
    
    # 3. V2X 通信量化 (D9)
    if 'v2x_comm' in quant_config:
        apply_comm_quantization(qmodel, quant_config['v2x_comm'])
    
    return qmodel
```

### A.3 敏感度分析工具

**新建文件**: `tools/sensitivity_analysis.py`

**功能**: 针对 D1-D9 的每个维度，自动化评估其对精度的影响

```
分析内容:

1. 逐层敏感度 (D7: 哪些层该量化)
   对每层 i: 仅量化该层 INT8, 其余 FP16 -> delta_AMOTA[i]

4. 校准方法对比 (D6: 用哪种校准)
   全局统一测 mse/minmax/entropy/percentile -> 选最优
   
5. 对称 vs 非对称 (D5)

6. 交互项分析 (D7: 层间耦合)

7. W-only vs W+A (D8: 量化什么)
   全局对比: 仅权重量化 vs 权重+激活量化

8. V2X 通信精度 (D9)
   对比: fp16 / int8 / int4 通信 -> 精度影响

输出: sensitivity_report.json (完整分析结果)
      -> 锁定不需搜索的维度, 缩减搜索空间
```

## Part B：TensorRT 侧部署端

> 目标：将 PyTorch 侧搜索得到的 quant_config 精确落地到 TRT 部署
> 核心机制：ONNX Q/DQ 显式量化模式
> 修改范围：`tools/inject_qdq_from_config.py`（新建）+ `tools/build_trt_int8_univ2x.py`（增强）

### B.1 PyTorch -> TRT 的维度映射

| 维度 | PyTorch 侧表达 | TRT 侧落地方式 |
|:----:|---------------|---------------|
| D1 w_bits | `weight_quantizer.n_bits` | Q/DQ scale 范围控制：INT8 scale=max/127, INT4 scale=max/7 |
| D2 a_bits | `act_quantizer.n_bits` | 同上，激活侧 Q/DQ scale 范围 |
| D3 w_granularity | `weight_quantizer.channel_wise` / `group_size` | Q/DQ scale tensor 维度：标量=per-tensor，向量=per-channel，分组向量=per-group |
| D4 a_granularity | `act_quantizer.channel_wise` | Q/DQ scale tensor 维度 |
| D5 symmetric | `weight_quantizer.sym` | Q/DQ zero_point：对称=0，非对称=非零（TRT GPU 仅支持 zp=0） |
| D6 scale_method | `weight_quantizer.scale_method` | PyTorch 侧选定方法算出 scale，通过 Q/DQ 节点传入 TRT |
| D7 layer_policy | 逐层 set_quant_state + bitwidth_refactor | 有 Q/DQ 的层走 INT8 kernel，无 Q/DQ 的层走 FP16 |
| D8 quant_target | set_quant_state(w, a) | W-only: 仅权重侧 Q/DQ；W+A: 权重+激活均有 Q/DQ |
| D9 comm_precision | CommQuantizer | 通信张量在传输前后插入 Q/DQ（自定义 ONNX 子图或 Python 侧处理） |

### B.2 TRT 侧对各维度的支持限制

| 维度 | TRT Q/DQ 支持情况 | 需要特殊处理 |
|:----:|:----------------:|-------------|
| D1/D2 位宽 | INT8 容器，通过 scale 范围模拟 INT4/INT6 | 无原生 INT4 kernel，走 INT8 kernel，有效精度由 scale 控制 |
| D3 per-group | Q/DQ 原生支持 per-channel，per-group 需要 reshape 技巧 | 权重 reshape 为 (out_ch*n_groups, group_size) 后 per-channel Q/DQ |
| D4 a_channel-wise | Q/DQ 支持 per-channel 激活 | 需要 axis 参数指定量化维度 |
| D5 非对称 | TRT GPU 不支持非零 zp | 强制使用对称（zp=0），非对称仅 DLA 支持 |
| D8 W-only | 仅权重 Q/DQ 会触发 explicit 模式导致 Calibrator 失效 | 必须同时插激活 Q/DQ（即便激活是 FP16 精度的 scale） |
| D9 comm | 不在 ONNX 模型图内 | 通信量化在 Python 推理侧处理，不涉及 TRT |

### B.3 Q/DQ 注入工具

**新建文件**: `tools/inject_qdq_from_config.py`

**输入**: FP32 ONNX + quant_config.json（含 PyTorch 校准的 scale 值）
**输出**: 带 Q/DQ 节点的 ONNX

**核心逻辑**:

```
对 quant_config.layers 中的每一层:

  if quant_target == "none":
    不插入 Q/DQ -> TRT 走 FP16
    continue

  # 权重侧 Q/DQ
  if "W" in quant_target:
    w_scale = 从 quant_config 读取 (PyTorch 校准结果)
    
    if w_granularity == "per_tensor":
      w_scale_tensor = scalar
    elif w_granularity == "per_channel":
      w_scale_tensor = vector (out_channels,)
    elif w_granularity == "per_group":
      # reshape 技巧: 将 weight (out, in) 视为 (out, n_groups, group_size)
      # per-group scale 展开为 per-channel scale on reshaped weight
      w_scale_tensor = vector (out_channels * n_groups,)
      insert_reshape_before_matmul(...)
    
    w_zp = zeros (对称)
    insert QuantizeLinear(weight_input, w_scale_tensor, w_zp)
    insert DequantizeLinear(...)

  # 激活侧 Q/DQ
  if "A" in quant_target:
    a_scale = 从 quant_config 读取 (PyTorch 校准结果)
    
    if a_granularity == "per_tensor":
      a_scale_tensor = scalar
    elif a_granularity == "per_channel":
      a_scale_tensor = vector, 需指定 axis
    
    a_zp = zeros (对称)
    insert QuantizeLinear(activation_input, a_scale_tensor, a_zp)
    insert DequantizeLinear(...)
  
  # 位宽模拟 (D1/D2):
  # scale 值本身已经编码了位宽信息:
  #   INT8: scale = max_abs / 127
  #   INT6: scale = max_abs / 31
  #   INT4: scale = max_abs / 7
  # Q/DQ 物理上用 INT8 容器, 但有效精度由 scale 范围决定

  # D8 W-only 的特殊处理:
  # 即使只量化权重, 也必须在激活侧插入一个 "透明" Q/DQ
  # (scale 极大, 相当于不量化, 但让 TRT 不退出 explicit 模式)
  if quant_target == "W-only":
    a_scale = very_large_value  # 模拟 FP16 精度的激活
    insert activation Q/DQ with large scale
```

**与之前失败方案的关键区别**:

| | 之��失败 (build_qdq_onnx_adaround.py) | 本次正确方案 |
|--|--------------------------------------|------------|
| 权重 Q/DQ | 有 | 有 |
| 激活 Q/DQ | **无** (根因) | **有** |
| 对称性 | 非对称 (TRT 不支持) | 对称 (zp=0) |

| 粒度 | 无控制 | per-tensor / per-channel / per-group |
| 位宽 | 仅 INT8 | INT4/INT6/INT8 通过 scale 范围模拟 |

### B.4 TRT 显式模式构建

**修改文件**: `tools/build_trt_int8_univ2x.py` 增加显式模式入口

```
新增命令行选项:
  --mode implicit    (现有隐式模式, Calibrator)
  --mode explicit    (新增显式模式, 读取 Q/DQ ONNX)

显式模式特点:
  - 不设置 Calibrator
  - 不调用 _force_msda_fp16() (由 Q/DQ 节点决定精度)

  - TRT 读取 Q/DQ scale, 自动 fuse 为 INT8 kernel
```

### B.5 Scale 导出工具

**新建功能**: 在 `quick_eval_quant.py` 中增加 scale 导出

```python
def export_scales_to_config(qmodel, quant_config):
    """
    PyTorch 校准完成后, 将每层的 scale 值写入 quant_config。
    这是 Part A -> Part B 的数据接口。
    """
    for name, module in qmodel.model.named_modules():
        if not isinstance(module, QuantModule):
            continue
        layer_cfg = quant_config['layers'].get(name)
        if layer_cfg is None or layer_cfg.get('quant_target') == 'none':
            continue
        
        # 权重 scale
        layer_cfg['w_scale'] = module.weight_quantizer.delta.detach().cpu().tolist()
        
        # 激活 scale
        if module.use_act_quant:
            layer_cfg['a_scale'] = module.act_quantizer.delta.detach().cpu().tolist()
    
    return quant_config
```
