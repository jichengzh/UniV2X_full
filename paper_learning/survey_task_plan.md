# 深度学习软硬件协同加速综述 — Multi-Agent协作任务书

> 项目发起: 2026-04-09
> 项目负责人: PM (主Agent, 即当前对话)
> 调研范围: 2023年至今
> 最终交付物: `paper_learning/survey_hw_sw_codesign.md`

---

## 1. 项目目标

撰写一份结构化的深度学习算法软硬件协同加速文献综述,覆盖2023年至今的前沿工作。要求:

- 论文数量 ≥ 30篇,每篇均可在网上查证(附arXiv/DOI链接)
- 优先顶会顶刊: ISCA, MICRO, HPCA, ASPLOS, DAC, ICCAD, DATE, NeurIPS, ICML, ICLR, CVPR, ECCV, MLSys, IEEE TCAD, IEEE JSSC 等
- 分类清晰,每篇论文包含: 标题、发表venue+年份、一句话核心贡献、协同方式关键词
- 单独章节覆盖自动驾驶/感知/车路协同方向的尝试
- **分类体系由Phase 0预调研确定,不预设**

---

## 2. 可用Agent资源

### 2.1 实际可调用的Agent类型

| Agent类型 | `subagent_type` | 核心能力 | 可用工具 | 本项目用途 |
|-----------|----------------|---------|---------|-----------|
| **通用型** | `general-purpose` (默认) | 网络搜索、文件读写、代码执行 | **全部工具**: WebSearch, Exa, Read, Write, Edit, Bash, Grep, Glob | 文献检索(唯一能做网络搜索的类型) |
| **探索型** | `Explore` | 快速代码库搜索 | Grep, Glob, Read (无WebSearch) | 不适用 |
| **规划型** | `Plan` | 架构规划 | Read, Grep, Glob (无WebSearch, 无Write) | 不适用 |
| **文档型** | `everything-claude-code:doc-updater` | 文档生成与更新 | Read, Write, Edit, Bash, Grep, Glob (无WebSearch) | Phase 3辅助撰写(可选) |

**关键约束: 能执行网络搜索的只有 `general-purpose` 类型。所有调研Agent必须使用此类型。**

### 2.2 Agent运行方式

| 特性 | 说明 |
|------|------|
| 并行启动 | 单条消息中发送多个Agent tool call,最多可同时启动4-5个 |
| 后台运行 | 设置 `run_in_background: true`,Agent完成后自动通知PM |
| 结果回传 | Agent返回纯文本结果给PM,PM负责写入文件 |
| 上下文隔离 | 每个Agent独立上下文,互相不可见,必须在prompt中提供完整指令 |
| 续传机制 | 可通过 `SendMessage` 向已完成的Agent追加任务 |

### 2.3 角色分配

| 角色 | 实现方式 | 职责 |
|------|---------|------|
| **PM** | 当前对话(主Agent) | 任务分发、结果汇总、去重、撰写最终报告、进程监控 |
| **预调研员** | 1个 `general-purpose` agent | Phase 0: 搜索survey论文和顶会session,确定分类体系 |
| **方向调研员 ×N** | N个 `general-purpose` agent (并行) | Phase 1: 按确定后的分类分别检索论文 |
| **文档撰写员** | `doc-updater` agent 或 PM自行撰写 | Phase 3: 辅助格式化最终报告(可选) |

N的数量和每个调研员的具体方向 **由Phase 0结果决定**。

## 3. 执行流程

### Phase 0: 预调研 — 确定分类体系

**目的:** 在实际调研之前,先摸清当前领域的实际方向划分,避免凭经验预设分类。

**执行方式:** 启动1个 `general-purpose` agent,执行以下任务:

```
任务: 调研2023-2026年深度学习软硬件协同加速领域的方向划分。

搜索内容:
1. 搜索该领域近两年的survey/综述论文,提取其分类体系
2. 搜索ISCA/MICRO/HPCA/DAC/ASPLOS 2024-2025的session/track名称中
   与DNN加速相关的主题
3. 搜索是否存在新兴协同方向(如LLM推理优化、存内计算CIM、
   光子/模拟计算、全栈协同框架等)

输出要求(≤800字):
1. 列出你发现的所有协同方向(含传统方向和新兴方向)
2. 每个方向一句话说明涵盖什么
3. 标注哪些方向论文密度高(≥10篇/年),哪些偏少
4. 推荐的分类方案(建议分几类,每类覆盖什么)
5. 列出你参考的survey论文(标题+链接)
```

**PM根据Phase 0输出决定:**
- 最终分类体系(几个大类,每类的边界)
- Agent数量和每个Agent的搜索范围
- 是否需要合并或新增方向
- 更新本任务书第4节的具体Agent任务定义

### Phase 1: 并行定向检索

**前置条件:** Phase 0完成,分类体系已确定。

```
PM ──┬── 分发任务 ──→ 调研Agent-1 (方向由Phase 0确定) ──→ 论文列表1
     ├── 分发任务 ──→ 调研Agent-2 (方向由Phase 0确定) ──→ 论文列表2
     ├── 分发任务 ──→ 调研Agent-3 (方向由Phase 0确定) ──→ 论文列表3
     ├── ...
     └── 分发任务 ──→ 调研Agent-N (自动驾驶/V2X, 固定) ──→ 论文列表N
```

**每个Agent的执行步骤:**
1. 使用WebSearch + Exa搜索相关论文(至少3轮不同关键词)
2. 筛选: 只保留2023年及以后、发表在目标venue的论文
3. 对每篇论文提取结构化信息(格式见第6节)
4. 按相关性排序,输出 ≥ 8篇
5. **严禁**输出搜索原始结果,只输出整理后的结构化论文列表

**所有调研Agent均使用 `general-purpose` 类型,通过prompt区分角色。**

### Phase 2: PM汇总与去重

PM收到全部Agent输出后:
1. 将各Agent结果分别写入 `paper_learning/survey_raw/agent_{1..N}.md`
2. 读取全部临时文件,执行去重(基于论文标题)
3. 验证每篇论文的链接格式完整性
4. 统计各分类论文数量,检查是否满足总数≥30
5. 如某方向不足,通过 `SendMessage` 向对应Agent追加搜索任务
6. 更新进程报告

### Phase 3: PM撰写最终报告

基于去重后的论文列表,PM撰写最终综述文档。报告结构在Phase 0之后确定,但固定包含:

```markdown
# 深度学习软硬件协同加速综述 (2023-2026)

## 1. 概述
  - 软硬件协同加速的定义与必要性
  - 本综述的范围与分类体系

## 2-M. [由Phase 0确定的各分类章节]
  每章包含:
  - 该方向的背景与核心问题
  - 论文列表(每篇按统一格式)

## M+1. 自动驾驶与车路协同中的实践 (固定章节)

## M+2. 趋势与总结
```

### Phase 4: 质量检查

- 逐篇验证论文链接可访问性(可用Bash curl检查)
- 格式统一性检查
- 分类合理性审核
- 交付最终文档

## 5. 上下文管理策略

### 5.1 Agent输出控制

| 控制点 | 规则 | 原因 |
|-------|------|------|
| 单篇论文描述 | ≤ 100字 | 防止单篇论文占用过多token |
| 单Agent总输出 | ≤ 1500字 | 防止Agent结果撑爆PM上下文 |
| 输出格式 | 仅结构化论文条目 | 搜索过程描述无信息价值 |
| 禁止内容 | 搜索原始HTML、完整摘要、参考文献列表 | 噪声信息 |

### 5.2 PM端上下文控制

| 控制点 | 规则 |
|-------|------|
| Agent结果接收 | 收到后立即写入临时文件,不在上下文中长期保留 |
| 临时存储 | `paper_learning/survey_raw/agent_{0,1,...,N}.md` |
| 最终合并 | PM从临时文件读取并合并,逐章撰写 |
| 进程报告 | 写入 `paper_learning/survey_progress.md`,每Phase更新 |

### 5.3 搜索轮次控制

每个调研Agent内部:
- 第1轮: 宽泛关键词,获取方向概览
- 第2轮: 针对第1轮发现的子方向,精确关键词补充
- 第3轮: 针对特定venue搜索,补充遗漏的重要工作
- 每轮搜索后立即筛选并丢弃不相关结果

## 6. 论文条目格式

所有Agent和最终报告统一使用:

```markdown
### [编号] 论文标题
- **Venue:** ISCA 2025
- **链接:** [arXiv](https://arxiv.org/abs/xxxx.xxxxx) 或 [DOI](https://doi.org/xxx)
- **协同维度:** 加速器架构 × 层融合 × 算子映射
- **简介:** 2-3句话。第一句说做了什么,第二句说怎么做的,第三句说效果。
```

## 7. Agent Prompt模板

### 7.1 预调研Agent Prompt (Phase 0)

```
你是一名深度学习硬件加速领域的研究分析师。

任务: 调研2023-2026年深度学习软硬件协同加速领域目前存在哪些研究方向。
不是找具体论文,而是确定方向划分。

请执行:
1. 搜索该领域近两年(2024-2026)的survey/综述论文(至少找3篇),
   提取它们的分类体系
2. 搜索ISCA, MICRO, HPCA, DAC, ASPLOS 2024-2026年的
   accepted paper list或program中与DNN/AI加速相关的session名称
3. 关注是否存在以下新兴方向,如果存在请标注:
   - LLM/大模型推理的软硬件协同
   - 存内计算(CIM/PIM)协同优化
   - Chiplet/异构计算协同
   - 全栈式端到端协同框架
   - 其他你发现的新方向

输出要求(≤800字):
1. 列出你发现的所有协同方向(每个方向一句话说明)
2. 标注论文密度: 高(≥10篇/年) / 中(5-10篇) / 低(<5篇)
3. 你推荐的分类方案(几个大类,每类边界)
4. 列出你参考的survey论文(标题+链接,≤5篇)
```

```
你是一名深度学习[方向名称]方向的文献调研员。

搜索策略:

交付要求:
- 找到 ≥ 8篇论文

### 论文标题
- **Venue:** xxx 20xx
- **链接:** https://...
- **协同维度:** A × B × C
- **简介:** 2-3句话

- 总输出 ≤ 1500字
- 不要包含搜索过程描述,只输出最终论文列表
```

## 9. 质量控制标准

### 9.1 论文筛选标准

**必须满足:**
- 发表年份 ≥ 2023
- 涉及软硬件协同(不能是纯算法或纯硬件)
- 有可查的arXiv链接或DOI
- 发表在认可的venue

**优先选择:**
- 被引次数较高的工作
- 开源代码的工作
- 知名研究组的工作(MIT Song Han, Georgia Tech Tushar Krishna, Stanford, 清华等)

### 9.2 分类标准

- 每篇论文归入**一个主分类**,可标注跨类关键词
- 分类依据论文的**核心贡献**,非涉及范围
- 具体分类边界由Phase 0确定

### 9.3 反幻觉措施

- Agent输出的每篇论文,PM阶段需验证:
  1. 论文标题在Google Scholar/arXiv可查
  2. 链接格式正确(arXiv ID格式或DOI格式)
  3. 发表venue与Agent标注一致
- 无法验证的论文标注"待验证"或删除

## 10. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| Phase 0发现新兴方向需要额外Agent | Agent数量超预期 | 合并相近方向,控制Agent总数≤5 |
| 某方向论文不足8篇 | 总数不达标 | 放宽venue至workshop或高质量arXiv |
| Agent输出幻觉论文 | 报告可信度下降 | PM逐篇验证(见9.3) |
| 论文跨类严重 | 分类模糊 | 允许交叉引用,但只在主分类下详细介绍 |
| 自驾方向软硬件协同论文稀少 | 该章节单薄 | 扩展到"部署优化"范畴 |
| 上下文溢出 | Agent输出被截断 | 严格执行1500字限制 |
| 并行Agent数量受限 | Phase 1执行变慢 | 分两批启动,先3个后2个 |
